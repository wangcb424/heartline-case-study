## Overview
This is a standalone UX case study website for **Heartline**, a course project focused on helping Northeastern students build real-life friendships with less awkwardness and emotional risk. The site documents the full design process (needfinding → lo-fi → hi-fi → testing → iteration → next steps) and is linked from my main portfolio.

---

## Requirement 1 — Project case study
**How it is met:**
This website includes a dedicated case study that goes beyond final screens and walks through:
- **Problem & POV** (target user Alex; why in-person initiation feels intimidating)
- **Needfinding / task framing** (what emotional and usability barriers we designed for)
- **Low-fidelity prototyping** (paper screens + a gallery of lo-fi artifacts)
- **High-fidelity prototyping** (embedded Figma prototype)
- **User research findings** (key hesitation points, discoverability issues, privacy concerns)
- **Iteration & improvement directions** (design changes and what we would do next)

**Evidence included on the site:**
- Lo-fi gallery images and captions
- Clear sections for findings and iteration decisions
- A “What’s Next” section describing future improvements (privacy-first location, onboarding, adoption strategy)
